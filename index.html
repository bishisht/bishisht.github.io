VisitMeet — A Privacy-Conscious Social Connection Platform

Overview:

VisitMeet is a web application built using Ruby on Rails that helps people connect based on their professions and locations, without revealing personal contact details. Inspired by the need for a simpler, safer way to meet like-minded individuals—especially while traveling—VisitMeet allows users to discover and engage with others nearby through shared interests or professional backgrounds.

Key Features:

  • Users can tag themselves with profession and location tags.
  • Login with email or username for flexible authentication.
  • Focus on privacy: no phone numbers or emails are exposed.
  • Built-in CAPTCHA protection using reCAPTCHA v3 to prevent spam.
  • Designed with a clean, user-friendly interface.

Tech Stack:

  • Ruby on Rails 8 (updated from Rails 7.1)
  • PostgreSQL
  • Importmap for JavaScript asset management
  • Stimulus.js for reactive UI
  • Responsive design with custom CSS

Current Status:

Under active development with a focus on refining user experience and strengthening privacy features. Planned features include tag-based search, activity preferences, and regional discovery.

Self-Hosted Linux Server with Dockerized Applications, Monitoring, and Security

Project Overview:

Deployed and managed a self-hosted Ubuntu Server to demonstrate end-to-end infrastructure setup and maintenance using modern DevOps practices. This system hosts multiple production-grade applications using Docker, with Nginx for reverse proxy, system hardening tools, and automation scripts for reliability and performance.

Infrastructure Setup:

  • Operating System: Ubuntu Server (Minimal Install)
  • Access Control: SSH with root login (secured via Fail2Ban and UFW)
  • Container Runtime: Docker + Docker Compose
  • Web Server: Nginx configured as a reverse proxy for containerized applications

Deployed Applications (via Docker Compose):

  • Uptime Kuma: Uptime monitoring dashboard for internal and external services/devices
  • NetBox: Infrastructure resource modeling and IP address management
  • EspoCRM: Lightweight CRM for multi-user access and container scalability

Security & Hardening:

  • UFW (Uncomplicated Firewall): Rules configured to allow only essential traffic (SSH, HTTP/HTTPS)
  • Fail2Ban: Automated blocking of IPs with multiple failed login attempts

Automation & Maintenance:

  • Maintenance Shell Script (Weekly via Cron): Tasks automated for log management and system cleanup.
  • Execution Time: Every Tuesday at 2:00 AM via crontab

Monitoring & Notifications:

  • Uptime Kuma Webhooks: Configured to send service downtime alerts through webhooks to Telegram

Tools & Technologies Used:

  • OS & Access: Ubuntu Server, SSH
  • Containers: Docker, Docker Compose
  • Reverse Proxy: Nginx
  • Monitoring: Uptime Kuma
  • CRM: EspoCRM
  • IPAM & Infra Doc: NetBox
  • Automation: Cron, Bash scripting
  • Security: UFW, Fail2Ban
  • Alerting: Webhook integrations
  • Version Control: Git

Outcome & Impact:

  • Built a secure, minimal server capable of running multiple Dockerized applications efficiently.
  • Demonstrated ability to provision, deploy, monitor, and maintain services in a resource-constrained environment.
  • Developed operational scripts and integrated reverse proxy and logging mechanisms.
  • Gained hands-on experience relevant to DevOps, infrastructure automation, and self-hosted solutions.
